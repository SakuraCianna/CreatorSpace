# CreatorSpace 需求文档

版本: v0.1 初稿  
日期: 2026-06-16  
状态: 待讨论 / 待确认  

## 1. 项目定位

CreatorSpace 是一个基于 Spring Boot 与 Vue 3 的个人主题风格博客与创意作品展示平台。它不只承担文章发布功能，还要作为个人品牌展示、创意项目陈列、内容管理和数据化运营的综合内容系统。

平台核心方向是“内容系统”，不是单一博客 CRUD。系统需要同时支持文章、作品集、主题配置、创意墙、评论互动、访问统计、文件资源、搜索和后台 CMS 管理。

- Vite 官方文档要求 Node.js `20.19+` 或 `22.12+`，当前 Node 24 满足要求。参考: https://vite.dev/guide/
- Vue 官方快速开始同样要求 Node.js `^20.19.0 || >=22.12.0`。参考: https://vuejs.org/guide/quick-start

## 3. 产品目标

### 3.1 核心目标

1. 建立一个可长期维护的个人内容平台。
2. 支持文章、作品、灵感、页面配置和主题风格统一管理。
3. 提供漂亮、可切换风格的访客前台。
4. 提供完整后台 CMS，让内容创建、管理、统计和配置都可视化完成。
5. 预留高级能力扩展空间，例如 Redis 统计、文章版本、搜索系统、AI 辅助写作和部署自动化。

### 3.2 非目标

初版不追求一次性完成所有高级功能。以下内容可以放到后续阶段:

- AI 写作、AI 摘要、AI 标签生成。
- 全文搜索引擎独立部署，例如 MeiliSearch 或 Elasticsearch。
- WebSocket 实时通知。
- 多租户站点。
- 复杂会员体系或商业支付。

## 4. 用户角色

| 角色 | 说明 | 核心权限 |
| --- | --- | --- |
| 游客 | 未登录访客 | 浏览公开内容、搜索、评论或留言 |
| 管理员 | 站点拥有者 | 管理所有内容、配置、主题、文件、评论和统计 |
| 编辑者 | 可选角色 | 管理文章、作品和创意卡片 |
| 友人 | 可选角色 | 查看部分私密内容 |

第一版可以只实现游客和管理员，编辑者与友人作为权限模型预留。

## 5. 前台访客端需求

### 5.1 首页

首页需要体现个人品牌和内容入口，建议包含:

- 头像、昵称、一句话介绍。
- 精选文章。
- 精选作品。
- 最近动态或创作时间线。
- 技术栈标签。
- 标签云。
- 访问统计摘要。
- 社交链接或联系方式。

首页内容应由后台配置，不建议硬编码在前端。

### 5.2 博客文章页

文章页需要支持:

- 文章列表。
- 分类筛选。
- 标签筛选。
- 关键词搜索入口。
- 推荐 / 置顶文章。
- 草稿不对外展示。
- 私密文章需要访问控制。

### 5.3 文章详情页

文章详情页需要支持:

- Markdown 渲染。
- 代码高亮。
- 文章目录。
- 封面图。
- 摘要。
- 分类与标签展示。
- 阅读量统计。
- 点赞。
- 评论列表。
- 上一篇 / 下一篇。

### 5.4 作品集页

作品集页用于展示个人项目、设计、摄影、视频或实验作品，需要支持:

- 作品列表。
- 类型筛选。
- 技术栈筛选。
- 推荐作品。
- 作品封面。
- GitHub 链接。
- Demo 链接。
- 视频链接。
- 展示 / 隐藏状态。

### 5.5 作品详情页

作品详情页需要支持:

- 项目简介。
- 技术栈标签。
- 项目截图。
- 开发过程。
- 遇到的问题与解决方案。
- 项目时间线。
- 外部链接。

### 5.6 创意墙

创意墙用于展示碎片灵感、Prompt、图片、UI 截图、短代码、链接或项目想法。页面形态建议接近瀑布流或 Gallery。

需要支持:

- 卡片列表。
- 卡片类型筛选。
- 标签筛选。
- 图片 / 文本 / Prompt / 代码 / 链接类型。
- 来源链接。
- 是否公开。

### 5.7 关于页

关于页需要支持:

- 个人介绍。
- 技能栈。
- 经历。
- 联系方式。
- 社交链接。
- 可选简历下载入口。

内容应优先来自后台页面配置。

### 5.8 留言板 / 评论互动

需要支持:

- 游客留言。
- 文章评论。
- 作品评论。
- 回复评论。
- 评论审核后展示。
- 评论点赞。

初版可以先做一级评论和审核，后续再扩展多级评论。

### 5.9 搜索页

搜索范围建议包括:

- 文章标题。
- 文章摘要。
- 文章内容。
- 作品标题。
- 作品描述。
- 创意卡片标题。
- 标签。
- 分类。

第一版可以使用 PostgreSQL 的基础模糊搜索。后续再评估 PostgreSQL 全文搜索、pg_trgm、MeiliSearch 或 Elasticsearch。

## 6. 后台 CMS 需求

### 6.1 后台登录

后台需要独立登录入口。第一版建议只开放管理员账号登录。

需要支持:

- 用户名 / 密码登录。
- JWT 或服务端会话方案待定。
- 登录状态过期处理。
- 未登录时禁止访问后台接口。

### 6.2 仪表盘

后台首页需要提供运营概览:

- 文章总数。
- 作品总数。
- 评论总数。
- 今日访问量。
- 最近 7 天访问趋势。
- 热门文章 Top 5。
- 热门作品 Top 5。
- 最新留言 / 评论。
- 标签分布。

### 6.3 文章管理

需要支持:

- 新建文章。
- 编辑文章。
- 删除文章。
- 草稿保存。
- 发布文章。
- 取消发布。
- 定时发布, 后续阶段实现。
- 置顶文章。
- 推荐文章。
- 私密文章, 后续阶段实现。
- Markdown 编辑。
- 封面上传。
- 分类与标签绑定。
- 文章版本保存与回滚, 后续阶段实现。

### 6.4 分类与标签管理

分类需要支持:

- 新增、编辑、删除。
- 排序。
- 启用 / 禁用。
- 用于文章或作品的类型区分。

标签需要支持:

- 新增、编辑、删除。
- 标签颜色。
- 标签权重或排序。
- 文章、作品、创意卡片复用。

### 6.5 作品管理

需要支持:

- 新增作品。
- 编辑作品。
- 删除作品。
- 设置展示 / 隐藏。
- 设置推荐。
- 上传封面。
- 管理项目截图。
- 配置 GitHub、Demo、视频等链接。
- 维护技术栈。
- 维护项目详情 Markdown。

### 6.6 创意墙管理

需要支持:

- 新增创意卡片。
- 编辑创意卡片。
- 删除创意卡片。
- 设置公开 / 私密。
- 上传图片。
- 维护来源链接。
- 选择卡片类型。
- 绑定标签。

### 6.7 评论管理

需要支持:

- 查看评论列表。
- 按目标类型筛选。
- 审核通过。
- 驳回。
- 删除。
- 拉黑昵称 / 邮箱 / IP, 后续阶段实现。
- 敏感词过滤, 后续阶段实现。

### 6.8 主题管理

主题系统是本项目特色能力。后台需要支持:

- 查看主题列表。
- 切换当前主题。
- 编辑主题基础信息。
- 配置主色、背景、字体、卡片样式和布局类型。
- 保存扩展 JSON 配置。
- 预览主题效果, 后续阶段实现。

预设主题建议:

| 主题 | 风格 |
| --- | --- |
| Cyber Night | 暗色科技风 |
| Soft Blog | 温柔浅色风 |
| Pixel World | 像素复古风 |
| Minimal White | 极简白色风 |
| Glass Space | 玻璃星空风 |

### 6.9 页面配置

页面配置用于避免把个人信息硬编码在前端。

需要支持:

- 站点名称。
- 站点副标题。
- 头像。
- 首页介绍文案。
- 社交链接。
- SEO 标题。
- SEO 描述。
- 首页精选内容配置。
- 关于页内容。

### 6.10 文件资源管理

需要支持:

- 上传图片。
- 上传文章封面。
- 上传作品封面。
- 上传作品截图。
- 文件列表。
- 文件预览。
- 文件分类。
- 文件大小限制。
- 文件引用统计, 后续阶段实现。
- 本地存储 / 对象存储方案待确认。

### 6.11 访问统计

需要支持:

- 记录访问路径。
- 记录访问目标类型。
- 记录 IP。
- 记录 User-Agent。
- 记录 Referer。
- 统计 PV。
- 统计 UV。
- 热门文章。
- 热门作品。
- 最近 7 天 / 30 天趋势。

第一版可以先写入 PostgreSQL。后续使用 Redis 做计数缓冲，再定时同步到 PostgreSQL。

### 6.12 操作日志

后台关键操作需要留痕:

- 新增文章。
- 更新文章。
- 删除文章。
- 发布文章。
- 切换主题。
- 审核评论。
- 上传文件。

后续可以通过 AOP 注解统一记录操作日志。

建议前端按前台与后台分区:

```text
src
├── app
├── router
├── stores
├── api
├── layouts
│   ├── public
│   └── admin
├── views
│   ├── public
│   └── admin
├── components
├── composables
```

前台重点是内容展示和主题风格。后台重点是高效管理，不要做成营销页，应该偏信息密度高、可扫描、可重复操作的 CMS 工作台。

## 11. 分阶段范围

### 11.1 第一阶段: 基础可运行版

目标: 前后端项目能完整跑起来，并具备核心内容发布能力。

建议范围:

- 后端项目初始化。
- 前端项目初始化。
- PostgreSQL 连接。
- Redis 连接预留或健康检查。
- 管理员登录。
- 文章发布。
- 文章列表。
- 文章详情。
- 分类与标签。
- 作品展示。
- 文件上传基础能力。
- 后台文章 / 作品管理。

不建议第一阶段加入:

- AI。
- 独立搜索引擎。
- 多角色复杂权限。
- WebSocket。
- 完整访问统计同步。

### 11.2 第二阶段: 完整内容平台

目标: 从基础博客扩展为内容展示与互动平台。

建议范围:

- 评论系统。
- 留言板。
- 主题切换。
- 页面配置。
- 数据看板。
- 访问统计。
- 作品详情页。
- 创意墙。
- 友情链接。

### 11.3 第三阶段: 后端技术亮点

目标: 增强项目技术含量和可讲述性。

建议范围:

- Redis 热门文章缓存。
- 访问计数 Redis 缓冲与定时同步。
- 定时发布。
- AOP 操作日志。
- 文章版本管理。
- 私密文章。
- 文件对象存储。
- PostgreSQL 全文搜索或独立搜索引擎。

### 11.4 第四阶段: AI / 全栈加分项

目标: 让平台具备更强的创作辅助能力。

建议范围:

- AI 生成摘要。
- AI 生成标签。
- AI 优化标题。
- AI 生成作品介绍。
- AI 搜索问答。
- WebSocket 通知。
- Docker 部署文档与脚本。

## 12. 非功能需求

### 12.1 安全

- 后台接口必须鉴权。
- 管理员密码必须加密存储。
- 文件上传必须校验大小和类型。
- 评论内容需要防 XSS。
- Markdown 渲染需要处理 HTML 注入风险。
- 不在 README、日志、提交信息中暴露密钥。

### 12.2 性能

- 首页和文章列表需要分页。
- 热门内容统计后续使用 Redis 降低数据库压力。
- 文件资源需要合理缓存。
- 后台数据看板需要避免每次全表扫描。

### 12.3 可维护性

- 前后端类型定义清晰。
- 后端统一响应结构。
- 后端统一异常处理。
- 后端模块边界清晰。
- 前端 API 调用集中管理。
- 文档中的命令必须适配 Windows PowerShell。

### 12.4 可观测性

- 后端需要基础日志。
- 关键后台操作需要操作日志。
- 后续可加入健康检查接口。
- 后续可加入接口耗时统计。

## 13. 验收标准

### 13.1 第一阶段验收

- 可以在 Windows 11 + PowerShell 环境启动后端。
- 可以在 Node 24 环境启动前端。
- 后端可以连接 PostgreSQL。
- 后端可以连接或检查 Redis。
- 管理员可以登录后台。
- 管理员可以新增、编辑、发布文章。
- 前台可以查看文章列表和详情。
- 管理员可以新增作品。
- 前台可以查看作品列表。
- 文件上传可用。
- README 说明启动方式、环境变量和基础命令。

### 13.2 第二阶段验收

- 前台主题可由后台切换。
- 首页内容可由后台配置。
- 评论提交后进入审核流程。
